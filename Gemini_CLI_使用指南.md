# Gemini CLI 使用指南 (新手入门版)

## 前言

欢迎使用 Gemini CLI（命令行界面）！这个指南旨在帮助你快速上手，了解如何与我（AI 助手）互动，以及如何利用 CLI 的各种功能来完成你的任务。即使你已经安装了 Gemini CLI，但对具体使用方法感到困惑，本指南也能为你提供清晰的指引。

---

## 1. 核心交互命令：`/` 和 `!`

在 Gemini CLI 中，与我互动主要通过两种特殊的命令前缀：`/` 和 `!`。

### 1.1 `/` 命令 (斜杠命令) - 探索我的“能力”

*   **作用**: 当你在命令行中输入 `/` 符号时，它会触发并显示我（AI 助手）当前**可以使用的所有工具列表**。这些工具代表了我的各种能力和功能，比如文件操作、代码修改、网络搜索等。
*   **何时使用**:
    *   当你不知道我能做什么，或者想了解我有哪些功能时。
    *   当你需要寻找特定功能，但不确定工具名称时。
*   **如何使用**:
    1.  在命令行中输入 `/`。
    2.  通常，界面会自动弹出可用的工具列表，或者你可以按一下 `Tab` 键来触发显示。
    3.  你可以开始输入工具名称的一部分来实时筛选列表，找到你需要的工具。
*   **示例**:
    *   输入 `/`，你可能会看到 `list_directory`, `read_file`, `write_file` 等工具。
    *   如果你输入 `/read`，列表可能会筛选出 `read_file`。

### 1.2 `!` 命令 (感叹号命令) - 执行我的工具

*   **作用**: `!` 符号用来**执行我（AI 助手）的某个工具**。这是你让我去执行特定任务的方式。
*   **何时使用**:
    *   当你已经知道要使用哪个工具及其参数时。
    *   当你需要立即让我执行某个具体操作时。
*   **如何使用**:
    1.  输入 `!`。
    2.  紧接着输入你要执行的工具的名称。
    3.  然后，输入该工具所需的任何参数（参数之间用空格分隔）。
*   **示例**:
    *   **执行文件读取**: `!read_file file_path=my_document.md`
        *   这会调用 `read_file` 工具，并告诉它读取 `my_document.md` 文件。
    *   **执行目录列表**: `!list_directory dir_path=./src`
        *   这会调用 `list_directory` 工具，列出 `./src` 目录下的内容。

*   **特别说明：运行 Shell 命令的简写**
    *   Gemini CLI 对运行标准的 Shell 命令提供了特别的简写。当你想运行一个系统命令（比如 `ls`、`cd`、`grep` 等）时，可以直接使用 `!` 加上命令。
    *   **例如**:
        *   `!ls -l` (等同于 `!run_shell_command command="ls -l"`)
        *   `!cd my_project` (等同于 `!run_shell_command command="cd my_project"`)
        *   `!grep "error" log.txt` (等同于 `!run_shell_command command="grep 'error' log.txt"`)
    *   这是因为 `run_shell_command` 是一个常用工具，这种简写方式大大提高了便利性。

## 2. 常用快捷键和操作技巧

熟悉这些快捷键可以显著提高你在 CLI 中的工作效率。

### 2.1 文本输入与导航

*   **`Tab` 键**:
    *   **命令补全**: 在输入命令或工具名称时，按 `Tab` 键可以自动补全，避免拼写错误。
    *   **路径补全**: 在输入文件或目录路径时，按 `Tab` 键可以补全路径。
    *   **参数建议**: 在某些情况下，输入工具名后按 `Tab` 键可能会显示可用的参数。
*   **`↑` (上箭头) / `↓` (下箭头) 键**:
    *   浏览你之前输入过的历史命令。这对于重复执行或修改之前的命令非常有用。
*   **`Ctrl + A` / `Home`**: 将光标移动到当前行的开头。
*   **`Ctrl + E` / `End`**: 将光标移动到当前行的末尾。
*   **`Ctrl + K`**: 从光标当前位置删除到行尾。
*   **`Ctrl + U`**: 从光标当前位置删除到行首。
*   **`Ctrl + W`**: 删除光标前的一个单词。

### 2.2 终端控制

*   **`Ctrl + C`**:
    *   **中断当前操作**: 当一个命令或操作正在执行，而你想停止它时，按 `Ctrl + C` 可以尝试中断它。
*   **`Ctrl + L` / `clear` 命令**:
    *   **清屏**: 清除终端屏幕上显示的所有内容，让你有一个干净的界面。你也可以直接输入 `!clear` 命令。

### 2.3 多行输入 (特殊情况)

在某些情况下，你可能需要输入包含多行文本的命令或信息。这在不同的终端环境和配置中行为可能不同：

*   **默认行为**: 通常情况下，在 CLI 中按下 `Enter` 键就会提交你的输入。
*   **尝试换行不提交**:
    *   如果你需要输入多行文本而不立即提交，可以尝试 **`Shift + Enter`** 或 **`Alt + Enter`**。这取决于你的终端模拟器和其配置。
    *   如果上述快捷键无效，并且 `Enter` 总是提交，你可以考虑将多行文本作为单个字符串参数传递，例如用三引号包裹（如果工具支持）。

## 3. 有效使用 Gemini CLI 的技巧

*   **阅读工具描述**: 当你使用 `/` 命令查看工具列表时，花时间阅读每个工具的简短描述，了解其功能。
*   **善用 `Tab` 补全**: 这是最能提高效率的习惯之一。
*   **从小处着手**: 先尝试简单的文件查看或目录列表命令，逐步熟悉流程。
*   **遇到问题及时提问**: 如果你不确定某个工具如何使用，或者遇到了错误，随时向我提问。
*   **利用我的记忆**: 我会记住你明确要求我记住的信息（比如你的开发角色、偏好等），这有助于我更好地理解你的意图并提供帮助。
