## 三、重要协议与概念

### 1. TCP 与 UDP 的区别

| 特性       | TCP (传输控制协议)          | UDP (用户数据报协议)     |     |
| :------- | :-------------------- | :---------------- | --- |
| **连接性**  | 面向连接                  | 无连接               |     |
| **可靠性**  | 可靠 (无差错、不丢失、不重复、按序到达) | 不可靠 (尽最大努力交付)     |     |
| **传输方式** | 面向字节流                 | 面向报文              |     |
| **拥塞控制** | 有                     | 无                 |     |
| **连接方式** | 只能点对点                 | 支持一对一、一对多、多对一、多对多 |     |
| **头部开销** | 20 字节                 | 8 字节              |     |
| **总结**   | 可靠，开销大，速度稍慢           | 不可靠，开销小，速度快       |     |

### 2. 地址解析协议 (ARP / RARP)

*   **ARP (Address Resolution Protocol)**: 将 **IP 地址** 转换为 **物理地址 (MAC 地址)**。
*   **RARP (Reverse Address Resolution Protocol)**: 将 **物理地址 (MAC 地址)** 转换为 **IP 地址**。

## 四、IP 地址

### 1. IPv4 地址分类

| 类别      | 最高位    | 网络地址范围                          | 说明                  |
| :------ | :----- | :------------------------------ | :------------------ |
| **A 类** | `0`    | `1.0.0.1` ~ `126.255.255.254`   | 1 字节网络地址 + 3 字节主机地址 |
| **B 类** | `10`   | `128.0.0.0` ~ `191.255.255.254` | 2 字节网络地址 + 2 字节主机地址 |
| **C 类** | `110`  | `192.0.1.1` ~ `223.255.255.254` | 3 字节网络地址 + 1 字节主机地址 |
| **D 类** | `1110` | `224.0.0.1` ~ `239.255.255.254` | 多播 (Multicast) 地址   |
| **E 类** | `1111` | -                               | 保留作将来使用             |

### 2. IPv6 的主要特点

*   **地址长度为 128 位**, 极大地扩展了地址空间。
*   **灵活的 IP 报文头部格式**，提高了处理效率。
*   **简化的报文头部格式**，字段仅 8 个，加快报文转发。
*   **提高安全性**，内置身份认证和隐私权保护。
*   **支持更多的服务类型**。
*   **允许协议继续演变**，方便增加新功能。
